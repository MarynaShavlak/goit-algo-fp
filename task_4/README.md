# Завдання 4 — Візуалізація бінарної купи

Бінарна купа зберігається як **масив**. Для елемента з індексом `i` його діти —
це елементи з індексами `2*i + 1` (лівий) і `2*i + 2` (правий). За цим правилом
будуємо з масиву бінарне дерево й малюємо його (`networkx` + `matplotlib`).

> Примітка завдання: суть — у створенні **дерева з купи**.

## Запуск

```bash
python task_4/main.py
```

Потрібні `networkx` і `matplotlib`.

## Як це працює

`build_heap_tree(heap, i=0)` рекурсивно будує дерево: створює вузол для
поточного індексу й рекурсивно — піддерева для `2*i + 1` та `2*i + 2`. За межами
масиву повертає `None`. Готовий корінь передається у `draw_tree`, який реалізує
розкладку (координати вузлів за рівнями) та малювання.

дітей.

## Аналіз рішення

### Сильні сторони

* Правильно використано індексні співвідношення `2*i + 1` / `2*i + 2` — це
  канонічний спосіб перейти від масиву-купи до дерева.
* Базовий код (`Node`, `add_edges`, `draw_tree`) повторно використано без змін
  логіки розкладки.

### Що покращено

1. **Розділено побудову й малювання.** У вихідному варіанті функція
   `render_heap_tree` водночас **будувала** дерево й **малювала** його як
   побічний ефект (`if i == 0: draw_tree(node)`). Це змішування відповідальностей.
   Тепер `build_heap_tree` лише будує дерево й **повертає корінь**, а малювання —
   окремий виклик `draw_tree(...)`. Так функцію можна перевикористати (наприклад,
   у Завданні 5 для обходів) без побічного малювання.

2. **Узгоджене створення вузлів.** У вихідному коді корінь створювався всередині
   (`if node is None`), а решта вузлів — батьком (`node.left = Node(...)` перед
   рекурсією). Тепер **кожен виклик створює свій вузол** і повертає його — логіка
   однакова для всіх рівнів і коротша.

3. **Порожня купа.** `heap[i]` для `i = 0` на порожньому списку давав би
   `IndexError`. Тепер перевірка `i >= len(heap)` повертає `None`, тож
   `build_heap_tree([])` коректно дає порожнє дерево.

4. **Демонстрація справжньої купи.** Окрім готового масиву, у демо
   показано побудову min-купи з довільних чисел через `heapq.heapify` — щоб
   наочно бачити, що візуалізується саме **купа**, а не довільний масив.

5. **Читабельні підписи.** Колір тексту вузла обирається за яскравістю фону
   (білий на темному, чорний на світлому) — корисно, якщо вузли розфарбовувати
   (як у Завданні 5).

## Складність (Big-O)

| Дія | Час | Пам'ять |
|---|---|---|
| `build_heap_tree` | O(n) | O(n) |
| `draw_tree` (розкладка + малювання) | O(n) | O(n) |

де `n` — кількість елементів купи. Кожен елемент масиву стає рівно одним вузлом
дерева й обробляється один раз.
