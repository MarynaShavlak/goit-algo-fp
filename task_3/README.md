# Завдання 3 — Дерева, алгоритм Дейкстри

Алгоритм Дейкстри для пошуку найкоротших шляхів у зваженому графі з
використанням **бінарної купи**. За приклад узято граф доріг між містами
України (відстані в км, приблизні); початкове місто — **Київ**.

## Запуск

```bash
python task_3/main.py
```

Потрібні `networkx` і `matplotlib`. У консоль виводяться відстані й маршрути, а
у вікні відкривається граф із виділеним деревом найкоротших шляхів.

## Як це працює

`dijkstra(graph, start)` тримає у словнику `distances` поточні найкоротші
відстані, а наступну вершину для обробки бере з **бінарної мін-купи** (`heapq`):
вершина з найменшою відомою відстанню витягується першою. Для кожного сусіда
виконується *релаксація* — якщо через поточну вершину дійти дешевше, відстань
оновлюється, у `previous` запам'ятовується попередник, і пара
`(відстань, вершина)` додається в купу.

`heapq` реалізує саме бінарну купу, тож `heappush` / `heappop` коштують
`O(log V)`, а весь алгоритм — `O((V + E) · log V)`.

## Аналіз рішення

### Сильні сторони

* Реалізація Дейкстри коректна: правильна ініціалізація відстаней,
  релаксація сусідів і вибір вершини з купи.
* Слушно застосовано *ліниве видалення* (`if current_distance > distances[u]:
  continue`) — застарілі записи у купі просто пропускаються, тож не потрібно
  видаляти елементи з середини купи.

### Що покращено

1. **Відновлення самих шляхів.** Вихідна функція повертала **лише відстані**,
   тоді як завдання просить «обчислення найкоротших шляхів … до всіх інших».
   Додано словник `previous` (попередник кожної вершини) і функцію
   `reconstruct_path`, яка відновлює маршрут, ідучи від цілі до старту.
2. **Чистота прикладу.** У вихідному коді ребро `D–E` додавалося двічі, а серед
   великих літер траплялася вершина `"j"`. У новому графі назви узгоджені (міста
   України), дублів немає.
3. **Структура коду.** Побудову графа, алгоритм, відновлення шляху та
   візуалізацію розділено на функції; запуск — під `if __name__ == "__main__"`.
4. **Яскравіша візуалізація.** Замість випадкової розкладки міста розставлені за
   приблизною географією України, а **дерево найкоротших шляхів** підсвічене
   кольором — одразу видно результат алгоритму, а не лише сам граф.

### Зауваження

* Алгоритм Дейкстри коректний лише для **невід'ємних** ваг ребер — для графів із
  від'ємними вагами потрібен інший алгоритм (наприклад, Беллмана–Форда).
* Недосяжні вершини залишаються з відстанню `∞`, а `reconstruct_path` повертає
  для них порожній список.

## Складність (Big-O)

| Операція | Складність |
|---|---|
| `heappush` / `heappop` (бінарна купа) | O(log V) |
| Алгоритм Дейкстри загалом | O((V + E) · log V) |

де `V` — кількість вершин, `E` — кількість ребер.
