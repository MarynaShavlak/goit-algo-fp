# Завдання 3 — Алгоритм Дейкстри на бінарній купі

Найкоротші шляхи у зваженому графі через бінарну мін-купу (`heapq`). За приклад —
граф доріг між містами України (відстані приблизні), старт із Києва.

## Запуск

`main.py` імпортує пакет `viz`, тож проєкт потрібно спершу встановити в
editable-режимі (див. [«Запуск» у кореневому README](../README.md#запуск)) —
інакше імпорт дасть помилку `ModuleNotFoundError: No module named 'viz'`:

```bash
pip install -e .
```

Потрібні `networkx` і `matplotlib`. У консоль у будь-якому режимі йдуть відстані
й маршрути.

**У вікні** — граф із підсвіченим деревом найкоротших шляхів (потрібне графічне
середовище):

```bash
python task_3/main.py
```

**У PNG** — через `matplotlib`, без дисплея (саме так зроблено `dijkstra.png`):

```bash
python task_3/main.py --save           # dijkstra.png поруч зі скриптом
```

## Як це працює

`dijkstra(graph, start)` зберігає поточні відстані у словнику `distances`, а
наступну вершину бере з купи — ту, до якої відома найменша відстань. Для кожного
сусіда робиться релаксація, і якщо знайдено коротший шлях, у `previous`
запам'ятовується попередник; з нього `reconstruct_path` згодом збирає маршрут.

Деталь, яку легко не помітити: у купі лишаються застарілі пари для вершини, до
якої вже знайдено коротший шлях. Замість видаляти їх із середини купи, такі
записи просто пропускаються (`if current_distance > distances[u]: continue`) —
це ліниве видалення. Складність — O((V + E)·log V).

Малювання графа винесене у `viz/dijkstra_graph.py`, тож у `main.py` лишилися
`build_graph`, `dijkstra` і `reconstruct_path`.

## Результат (від Києва)

| Місто | Відстань | Маршрут |
|---|---:|---|
| Київ | 0 | Київ |
| Вінниця | 270 | Київ → Вінниця |
| Полтава | 340 | Київ → Полтава |
| Одеса | 475 | Київ → Одеса |
| Харків | 480 | Київ → Полтава → Харків |
| Дніпро | 530 | Київ → Полтава → Дніпро |
| Львів | 540 | Київ → Львів |
| Запоріжжя | 615 | Київ → Полтава → Дніпро → Запоріжжя |

![Дерево найкоротших шляхів](dijkstra.png)

Помаранчевим — дерево найкоротших шляхів, сірим — решта доріг. До Харкова, Дніпра
й Запоріжжя вигідніше їхати через Полтаву: прямого ребра Київ–Харків немає, тож
найкоротший шлях становить 340 + 140 = 480 км.
