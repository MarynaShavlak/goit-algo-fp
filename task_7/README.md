# Завдання 7 — Метод Монте-Карло: суми двох кубиків

Кидаємо два кубики багато разів, рахуємо ймовірність кожної суми (2–12) і
порівнюємо з точними (аналітичними) значеннями.

## Запуск

```bash
python task_7/main.py
```

Ядро — стандартна бібліотека; для графіка потрібен `matplotlib` (без нього таблиця
все одно виводиться). Зберегти графік (`dice.png`):

```bash
python task_7/main.py --save
```

## Як це працює

`monte_carlo_dice(n)` кидає два кубики `n` разів і ділить лічильники сум на `n`.
Точні значення дає `analytical_probabilities`: для суми `s` число комбінацій —
`6 − |s − 7|` (від 1/36 для 2 і 12 до 6/36 для 7) із 36 рівноймовірних. Запуск
фіксує `random.seed(42)`, щоб числа в README збігалися з реальним прогоном.
Графік малює `viz/dice_chart.py`.

| 5 | 4/36 | 11.11% | 11.09% | 0.02 |
| 6 | 5/36 | 13.89% | 13.89% | 0.00 |
| 7 | 6/36 | 16.67% | 16.66% | 0.00 |

### Збіжність

Що більше кидків, то менше відхилення (приблизно як 1/√n):

| Кидків | Макс. відхилення |
|---:|---:|
| 1 000 | 1.489 в.п. |
| 10 000 | 0.263 в.п. |
| 100 000 | 0.219 в.п. |
| 1 000 000 | 0.04 в.п. |

Спадання не строго монотонне — метод стохастичний, числа стрибають від запуску до
запуску. Але тренд чіткий: щоб зменшити похибку вдесятеро, кидків треба приблизно
у сто разів більше.
