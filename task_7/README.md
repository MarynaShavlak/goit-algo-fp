# Завдання 7 — Метод Монте-Карло: суми двох кубиків

Два кубики кидаються багато разів; для кожної суми (2–12) обчислюється
ймовірність і порівнюється з точними (аналітичними) значеннями.

## Запуск

`main.py` імпортує пакет `viz`, тож проєкт потрібно спершу встановити в
editable-режимі (див. [«Запуск» у кореневому README](../README.md#запуск)) —
інакше імпорт дасть помилку `ModuleNotFoundError: No module named 'viz'`:

```bash
pip install -e .
```

Порівняльна таблиця завжди виводиться в консоль і тримається лише на стандартній
бібліотеці. `matplotlib` — м'яка залежність *тільки для графіка*: без нього
таблиця все одно друкується, а графік пропускається.

**У вікні** — графік у вікні (потрібне графічне середовище):

```bash
python task_7/main.py
```

**У PNG** — `--save` пише `dice.png` поруч зі скриптом (backend Agg, без
дисплея):

```bash
python task_7/main.py --save
```

## Як це працює

`monte_carlo_dice(n)` кидає два кубики `n` разів і ділить лічильники сум на `n`.
Точні значення дає `analytical_probabilities`: для суми `s` число комбінацій —
`6 − |s − 7|` (від 1/36 для 2 і 12 до 6/36 для 7) із 36 рівноймовірних. Запуск
фіксує `random.seed(42)`, щоб числа в README збігалися з реальним прогоном.
Графік малює `viz/dice_chart.py`.

### Збіжність

Що більше кидків, то менше відхилення (приблизно як 1/√n):

| Кидків | Макс. відхилення |
|---:|---:|
| 1 000 | 1.489 в.п. |
| 10 000 | 0.263 в.п. |
| 100 000 | 0.219 в.п. |
| 1 000 000 | 0.04 в.п. |

Спадання не строго монотонне — метод стохастичний, числа стрибають від запуску до
запуску. Але тренд чіткий: щоб зменшити похибку вдесятеро, кидків треба приблизно
у сто разів більше.
